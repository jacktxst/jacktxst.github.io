Sprites

All art for the game is packaged into a single sprites.png file. There is a value in the config, tileSize, which determines how sprites.png will be split up into individual sprites.

A sprite is referred to by an id number. Sprite 0 is the sprite located at the top left of sprites.png. Sprite 0 should typically be blank space.

A tile id is actually just a sprite id.

Although tileSize determines how sprites are indexed, you can easily draw sprites of varying sizes.

drawSprite(spriteId, dx, dy, width, height)

  • spriteId
  • dx - destination x coordinate, top left corner of the sprite on the screen
  • dy - destination x coordinate, top left corner of the sprite on the screen
  • w - width. can extend into neighboring cells on the sprite sheet to create sprites bigger than tileSize.
  • h - height. can extend into neighboring cells on the sprite sheet to create sprites bigger than tileSize.
  • r - rotation in radians

entity animations

entities need to support a few different types of animations. for now i'm going to implement these in the NPC entity type. through commands, there needs to be a way to setup animations where the sprite frame shown for an entity changes over time, and there needs to be a way for an npc to move along a scripted path. after these animations finish, commands can automatically be set to run. through this system the developer can create all kinds of scripted sequences, cutscenes etc. read the article about npcs for more info.

tile animations

the developer should be able to define animation data for every individual tile type, and those tiles should display as animated in the world. for example, running water.
