Collision System

A brief description of all features related to collision detection and handling in the engine.

Noclip Command

The .noclip command toggles collision for the player.

  • .noclip - Toggle collision on/off
  • .noclip 0 - Explicitly disable collision
  • .noclip 1 - Explicitly enable collision

When out of bounds, you can walk back in bounds.

Tile Collision

You should be able to run a command that sets the collision mode of a certain tile ID. Currently the engine just stores a hardcoded list of tile IDs which act like full-block walls. The engine also allows for collision to be enabled and disabled on a layer-per-layer basis, meaning that some layers of the map can include wall type blocks without preventing the player's movement. This feature is not yet accesible via commands.

Trigger Entity

The trigger entity is used to detect whenever the player enters a region. It can run a command whenever the player enters. Very useful!

World Bounds

The collision system handles world boundaries to prevent entities from moving outside the playable area.

Info Invisibility

The .info invis [0|1] command controls visibility of collision information in the debug display.

  • .info invis 0 - Show collision info
  • .info invis 1 - Hide collision info
