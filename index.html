Overview

pixlworldmaker is a lightweight, cross-platform, open-source web-based game engine for 2D games with pixellated graphics. It's particularly tailored in favor of RPG style games, but is capable of doing more. It has a hybrid editor/player architecture, so the developer can run commands to edit game content at runtime. The engine is designed to include enough built-in features for a game to be made only through the use of developer commands and scripting using a simple custom script system. I made this engine using vanilla JavsScript, HTML and CSS.

What is this for?

If you want to make a game like Undertale, Earthbound, A Link to The Past, Final Fantasy 6, this engine might be for you.

If you're an artist and don't want to worry about programming, you're definitely in the right place. The game making tools of the engine are designed to be very beginner friendly. Maybe not quite yet, but once I work on some more tutorials and examples.

If you want to make games that can run on any platform including mobile, this engine can do that very very well. (Not quite yet, but this will be easy to add)

If you want to make games that have really small sizes and run extremely fast even on the worst devices, this engine is made exactly for that!

Getting Started

I plan to offer a few different release flavors of the engine as well as some add-on modules for extra features. For now, the entire engine lives in a single .html file. All you have to do is download this file. You don't need a server to use it, it runs just fine locally. All the JavaScript code for the engine is in this file. For more complex games you'll likely want to use multiple JavaScript files. The engine will support this.

For real though, you'll probably want to download some more files from the GitHub repository to help get you started. The sections below explain the kinds of things to look for.

sprites.png

In order to start, the engine needs a sprites.png file. This file can be placed in the directory with the html file or can be dragged onto the page at any time. The sprites.png file contains ALL images for the game and is divided by the engine into a set of tiles / sprites of a certain height and width. Click here to learn more about sprites and sprites.png.

Currently, sprites.png is the only file that is automatically picked up by the engine if it's placed into the directory. This is because browsers don't let locally running web pages grab random files. Images are an exception to this. There are a lot of good ways to get around this which will eventually be supported by the engine.

.map file

A .map file stores the tile data of the entire world in an uncompressed binary format. Through a point-and-click interface combined with a set of commands like fill and copy, the map can be edited with ease in-game by the developer. The map can also easily be saved to a file. Click here to read more about the map and .map files.

When you publish a game, there will be support for placing .map files in the directory with the web page, but this is not yet supported. For now, you have to drag a .map file onto the game or run the command .import map.

scripts

A script is a series of commands that can be triggered by certain events and entities in the game. You can start writing a script in game using the .edit command, or you can drag a .txt file onto the game and it will immediately execute as a script. Some .txt file scripts are made to define several scripts without running them using the .edit command. This means that all the scripts for a game can be packaged into a single, or multiple .txt files. Scripts are responsible for the creation of entities , the management of U.I. and dialog, setting configuration values, and more. Click here to read more about scripts.

When you publish a game, there will be support for placing .txt files in the directory with the web page, but this is not yet supoprted.
